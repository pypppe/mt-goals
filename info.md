# Commit Starters Meaning

```feat:``` - A new feature visible to the user or consumer of the code.

```fix:``` - A bug fix.

```chore:``` - Routine maintenance that doesn't touch production logic. Things like updating dependencies, tweaking build configs, or cleaning up files.

```docs:``` - Documentation only. README changes, inline comments, wikis.

```refactor:``` - Code was restructured or cleaened up. But little to no changes to behavior. No new features no bug fixes.

```tests:``` - Testing website updates.

```style:``` - Code formatting only.

```perf:``` - A change that improves performance without altering/changing behavior.

```revert:``` - Reverting from a previous commit.

```oth:``` - Commits that do not fit any of the above.

```e-proj:``` - End a project. (e.g. Monkeytype Goals Posts v1)

```u-upd:``` - Remove something that is no longer needed/unnessecary in the code.

```gh-add:``` - Add a new file to the GitHub that doesn't affect the website.
